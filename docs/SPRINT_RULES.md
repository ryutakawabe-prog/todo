## 🏷️ ラベル体系

### スプリント
- `sprint-1` ～ `sprint-12` - どのスプリントに属するか

### 優先度
- `priority:high` - 即座に対応が必要
- `priority:medium` - スプリント内に対応
- `priority:low` - 次スプリント以降で OK

### ステータス
- `status:todo` - 未着手
- `status:in-progress` - 進行中
- `status:review` - レビュー待ち
- `status:done` - 完了

### タイプ
- `type:feature` - 新機能
- `type:bug` - バグ修正
- `type:improvement` - 改善・リファクタ
- `type:documentation` - ドキュメント

## 📝 Issue 作成時のルール

### タイトル形式
```
[タイプ] 機能概要 (SP: ストーリーポイント)
```

**例：** `[FEATURE] ユーザー登録機能を実装する (SP: 5)`

### 必須項目
- [ ] タイトルに ストーリーポイント (SP: X) を記載
- [ ] 適切なスプリント ラベルを付与
- [ ] 優先度 ラベルを付与
- [ ] Milestone を設定
- [ ] 受け入れ基準を記載

### 任意項目
- アサイン：担当者を指定
- Description：詳細説明、タスク分解など

## 🎯 スプリント進行の流れ

### 【スプリント開始時】
1. Sprint Planning で今スプリントのゴールを決定
2. Issues を作成・整理
3. Projects ボードに追加
4. 各タスクにアサイン

### 【スプリント進行中】
1. 毎日 Projects ボードを確認
2. 進捗に応じてステータスラベルを更新
3. 必要に応じてタスクを追加・削除

### 【スプリント終了時】
1. Sprint Review - 完了状況を確認
2. Sprint Retro - 改善点をまとめる
3. 次スプリント計画を開始

## 📊 Projects ボードの活用

Projects V2 の Kanban ボードで以下のようにステータス管理：

```
📝 To Do       → 🔄 In Progress → 👀 In Review → ✅ Done
```

### ビュー例
- **Board View** - スプリントの進捗を視覚的に確認
- **Table View** - Issues の詳細情報を表示
- **Roadmap View** - 複数スプリント全体の計画を表示

---

## 💡 Tips

- **ストーリーポイント**：タスクの相対的な複雑さ・工数を数値化（通常 1～13）
- **受け入れ基準**：タスク完了の条件を明確にすることで、手戻りを防止
- **定期ミーティング**：Sprint Planning, Daily Standup, Sprint Review, Retro を定期実施推奨
